# Privacy Policy

## Data Privacy and Security

The Nimble Streamer Log Analyzer is designed with privacy and data security as core principles.

## What We Collect

**We do not collect any data.** This application:

- ✅ **Runs entirely locally** on your machine
- ✅ **Does not send data** to external servers
- ✅ **Does not track users** or usage patterns
- ✅ **Does not store data** outside your local environment
- ✅ **Does not require registration** or personal information

## Your Log Data

### Local Processing
- All log file analysis happens **locally on your computer**
- No log data is transmitted over the internet
- Your server IPs, stream names, and log contents remain private
- All generated reports stay on your local machine

### Data Storage
- Log files are stored in your local `logs/` directory
- Analysis reports are saved to your local `reports/` directory  
- No data is stored in cloud services or remote databases
- You have complete control over your data

### Data Retention
- **You control data retention** - delete files when you no longer need them
- The application does not automatically delete or backup your data
- No data is retained by the developers or any third parties

## Web Interface Privacy

### Local Web Server
- The web interface runs on `127.0.0.1` (localhost only)
- **Not accessible from external networks** by default
- No data is transmitted outside your local machine
- Web interface is only for local analysis visualization

### Browser Data
- No cookies are set for tracking purposes
- No analytics or telemetry data is collected
- No external resources are loaded (all assets are local)

## Third-Party Libraries

This application uses open-source Python libraries:
- **pandas, matplotlib, plotly, dash** - For data processing and visualization
- **All processing happens locally** - these libraries do not transmit data externally
- Libraries are used only for local computation and display

## Security Measures

### Code Transparency
- ✅ **Open source** - All code is publicly available for review
- ✅ **No hidden functionality** - What you see is what you get
- ✅ **Community reviewed** - Code can be audited by security professionals

### Safe Practices
- No network requests to external services
- No execution of remote code
- Input validation and sanitization implemented
- Error handling prevents data leakage in logs

## Data Protection Compliance

### GDPR Compliance
- ✅ **No personal data collection** - GDPR does not apply
- ✅ **Local processing only** - No cross-border data transfers
- ✅ **User control** - You maintain complete control of your data

### Industry Standards
- Follows security best practices for local applications
- Uses established libraries with good security track records
- Regular updates to maintain security standards

## Recommendations for Users

### Protecting Your Data
1. **Keep log files secure** - Store in protected directories
2. **Limit access** - Only authorized personnel should access the application
3. **Regular cleanup** - Delete old reports and logs when no longer needed
4. **Network security** - Use firewall rules if needed for additional protection

### Sensitive Information
- The application is designed to handle sensitive server logs safely
- Server IPs and stream information remain on your local system
- Consider the sensitivity of your log data and handle accordingly

## Questions or Concerns

If you have privacy or security concerns:

1. **Review the source code** - All functionality is transparent and open
2. **File an issue** on our GitHub repository for security concerns
3. **Contact the maintainers** through GitHub for privacy questions

## Updates to This Policy

- This privacy policy may be updated to reflect application changes
- Updates will be communicated through GitHub releases
- Major privacy-related changes will be highlighted in release notes
